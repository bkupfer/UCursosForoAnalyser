Aidan Hogan 9 de Abril a las 13:40 hrs.
I'm really sorry that the lab didn't run yesterday. This time it wasn't firewalls or server configurations ... it was very much my fault. Before the lab I accidentally tested an old version of the code that worked, not the code with the new API. The WordCount code I gave you as an example in mdp-lab5.zip had two bugs that you would have copied into your new classes. Sorry. :/

But the good news is that the fixes are easy.

I will copy a working version of the WordCount with the new API below.

The main changes are listed below. If you make these changes to your new class files they should work. If not, feel free to leave me a message here.

Taking my mistake into account, there's no need to submit this lab until April 20th (a week delayed). I would however ask you to finish the coding for the lab before the next lab on Wednesday ... we can take the first 15 minutes to test it then.

Also I'll give a little competition: the first person to post correct answers here for both (1) the most frequent co-stars in IMDb and how many theatrical movies they appeared in together, (2) the number of theatrical movies Al Pacino and Robert De Niro have co-starred in ... I'll give them 2 bonus points for the overall lab score (each lab is worth 10).

== 1 ==

In the main method:

  1. job.submit();


Indeed submits the job but doesn't wait for it to complete ... it returns immediately. To keep the dialogue open, instead use:

  1. job.waitForCompletion(true);


== 2 ==

In the reduce method, the second argument needs to be Iterable, not Iterator. If you use Iterator, the reduce will silently not run and the output will just be the mapper (this took me like an hour of pulling out my hair to figure out ... grargh ... the reason why it doesn't show a compiler error is because the @Override annotation is in the blue comments, not directly above the method. If you put @Override above your map and reduce methods, you should get informative compiler errors.)

  1. job.setMapOutputKeyClass(Text.class);

  2.     job.setMapOutputValueClass(IntWritable.class);


== Other stuff ==

Unlike the WordCount example, you may need to explicitly set the types of the Mapper output key and value in the main method (you didn't need to do this before since the output types for the Mapper and Reducer were the same).

== Working WordCount example ==

  1. package org.mdp.hadoop.cli;

  2. 

  3. import java.io.IOException;

  4. 

  5. import org.apache.hadoop.conf.Configuration;

  6. import org.apache.hadoop.fs.Path;

  7. import org.apache.hadoop.io.IntWritable;

  8. import org.apache.hadoop.io.Text;

  9. import org.apache.hadoop.mapreduce.Job;

  10. import org.apache.hadoop.mapreduce.Mapper;

  11. import org.apache.hadoop.mapreduce.Reducer;

  12. import org.apache.hadoop.mapreduce.lib.input.FileInputFormat;

  13. import org.apache.hadoop.mapreduce.lib.output.FileOutputFormat;

  14. import org.apache.hadoop.util.GenericOptionsParser;

  15. 

  16. /**

  17.  * Java class to run a remote Hadoop word count job.

  18.  * 

  19.  * Contains the main method, an inner Reducer class 

  20.  * and an inner Mapper class.

  21.  * 

  22.  * @author Aidan

  23.  */

  24. public class WordCount {

  25. 	

  26. 	/**

  27. 	 * Use this with line.split(SPLIT_REGEX) to get fairly nice

  28. 	 * word splits.

  29. 	 */

  30. 	public static String SPLIT_REGEX = "[^\\p{L}'-]+";

  31. 	

  32. 	/**

  33. 	 * This is the Mapper Class. This sends key-value pairs to different machines

  34. 	 * based on the key.

  35. 	 * 

  36. 	 * Remember that the generic is Mapper<InputKey, InputValue, MapKey, MapValue>

  37. 	 * 

  38. 	 * InputKey we don't care about (a LongWritable will be passed as the input

  39. 	 * file offset, but we don't care; we can also set as Object)

  40. 	 * 

  41. 	 * InputKey will be Text: a line of the file

  42. 	 * 

  43. 	 * MapKey will be Text: a word from the file

  44. 	 * 

  45. 	 * MapValue will be IntWritable: a count: emit 1 for each occurrence of the word

  46. 	 * 

  47. 	 * @author Aidan

  48. 	 *

  49. 	 */

  50. 	public static class WordCountMapper extends Mapper<Object, Text, Text, IntWritable>{

  51. 

  52. 		private final IntWritable one = new IntWritable(1);

  53. 		private Text word = new Text();

  54. 

  55. 		

  56. 		/**

  57. 		 * @throws InterruptedException 

  58. 		 * @Override

  59. 		 * 

  60. 		 * This is the map method that you're goint to write. :)

  61. 		 */

  62. 		@Override

  63. 		public void map(Object key, Text value, Context output)

  64. 						throws IOException, InterruptedException {

  65. 			String line = value.toString();

  66. 			String[] rawWords = line.split(SPLIT_REGEX);

  67. 			for(String rawWord:rawWords) {

  68. 				if(!rawWord.isEmpty()){

  69. 					word.set(rawWord.toLowerCase());

  70. 					output.write(word, one);

  71. 				}

  72. 			}

  73. 		}

  74. 	}

  75. 

  76. 	/**

  77. 	 * This is the Reducer Class.

  78. 	 * 

  79. 	 * This collects sets of key-value pairs with the same key on one machine. 

  80. 	 * 

  81. 	 * Remember that the generic is Reducer<MapKey, MapValue, OutputKey, OutputValue>

  82. 	 * 

  83. 	 * MapKey will be Text: a word from the file

  84. 	 * 

  85. 	 * MapValue will be IntWritable: a count: emit 1 for each occurrence of the word

  86. 	 * 

  87. 	 * OutputKey will be Text: the same word

  88. 	 * 

  89. 	 * OutputValue will be IntWritable: the final count

  90. 	 * 

  91. 	 * @author Aidan

  92. 	 *

  93. 	 */

  94. 	public static class WordCountReducer extends Reducer<Text, IntWritable, Text, IntWritable> {

  95. 

  96. 		/**

  97. 		 * @throws InterruptedException 

  98. 		 * @Override

  99. 		 * 

  100. 		 * This is the reduce method that you're going to write. :)

  101. 		 */

  102. 		@Override

  103. 		public void reduce(Text key, Iterable<IntWritable> values,

  104. 				Context output) throws IOException, InterruptedException {

  105. 			int sum = 0;

  106. 			for(IntWritable value: values) {

  107. 				sum += value.get();

  108. 			}

  109. 			output.getCounter("words", key.toString().substring(0, 1)).increment(1);;

  110. 			output.write(key, new IntWritable(sum));

  111. 		}

  112. 	}

  113. 

  114. 	/**

  115. 	 * Main method that sets up and runs the job

  116. 	 * 

  117. 	 * @param args First argument is input, second is output

  118. 	 * @throws Exception

  119. 	 */

  120. 	public static void main(String[] args) throws Exception {

  121. 		Configuration conf = new Configuration();

  122. 		String[] otherArgs = new GenericOptionsParser(conf, args).getRemainingArgs();

  123. 		if (otherArgs.length != 2) {

  124. 			System.err.println("Usage: WordCount <in> <out>");

  125. 			System.exit(2);

  126. 		}

  127. 		String inputLocation = otherArgs[0];

  128. 		String outputLocation = otherArgs[1];

  129. 		

  130. 		Job job = Job.getInstance(new Configuration());

  131. 	     

  132. 	    FileInputFormat.setInputPaths(job, new Path(inputLocation));

  133. 	    FileOutputFormat.setOutputPath(job, new Path(outputLocation));

  134. 	    

  135. 	    job.setOutputKeyClass(Text.class);

  136. 	    job.setOutputValueClass(IntWritable.class);

  137. 	    job.setMapOutputKeyClass(Text.class);

  138. 	    job.setMapOutputValueClass(IntWritable.class);

  139. 	    

  140. 	    job.setMapperClass(WordCountMapper.class);

  141. 	    job.setCombinerClass(WordCountReducer.class);

  142. 	    job.setReducerClass(WordCountReducer.class);

  143. 	     

  144. 	    job.setJarByClass(WordCount.class);

  145. 		job.waitForCompletion(true);

  146. 	}	

  147. }

Hey Cristian. As Jaime says this week it'll be really tricky to test outside of the lab, but if you want to try, you can call StartRegistryAndServer to start a central directory on a second computer (we'll call C2) in the lab. The arguments you'll need are "-n [ip_of_C2] -r -sp -s 1".

Then you can follow the instructions of the lab on the first machine using the IP of C2 for "replace_me" and "[registry_location]" in the notes.

When you're finished, if you can message yourself without errors, then your code should be working.

Unfortunately you might have problems with the firewall (the sistemas turned it off for the lab). If so, another option is to run everything on the one machine but on different ports (you can use, e.g., "-p 1984" to change the port; use one port for the central directory and one for the messaging service).
